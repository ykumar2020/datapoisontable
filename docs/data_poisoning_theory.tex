\begin{abstract}
Data poisoning attacks manipulate the data used to train, fine-tune, align, retrieve for, or otherwise condition machine learning systems. This paper presents a risk-based framework for organizing poisoning and related adversarial methods into a ``Periodic Table of Data Poisoning.'' The framework separates training-time poisoning from test-time evasion, groups techniques by mechanism, and scores operational danger using impact, stealth, scalability, and cost-efficiency. Real MNIST and fungi image-classification experiments support the framework with executable comparisons across label poisoning, subpopulation poisoning, backdoor poisoning, clean-label feature collision, gradient-matching poisoning, and related evasion attacks. The results show that backdoors and subgroup-targeted poisoning can preserve or obscure aggregate clean accuracy while creating severe targeted failure modes, supporting the need for risk-aware evaluation beyond ordinary accuracy.
\end{abstract}

\begin{IEEEkeywords}
data poisoning, backdoor attacks, adversarial machine learning, fungi classification, MNIST, AI security, risk assessment
\end{IEEEkeywords}

\section{Introduction}

Machine learning systems are increasingly trained from data that is collected, labeled, filtered, retrieved, or reused through complex pipelines. These pipelines create an attack surface: an adversary may influence the data before training, fine-tuning, retrieval, preference optimization, or deployment-time adaptation. Data poisoning attacks exploit this surface by modifying a small part of the data so that the learned model behaves incorrectly.

The original poster project organized adversarial techniques into a visual ``Periodic Table.'' This paper strengthens that idea in three ways. First, it separates data poisoning from test-time evasion. Many well-known methods, including FGSM, PGD, JSMA, ZOO, SparseFool, HopSkipJump, and Adversarial Patch, modify inputs at inference time and should not be labeled as poisoning unless their outputs are inserted into the training set. Second, it introduces a risk model based on operational danger rather than mathematical elegance alone. Third, it provides executable MNIST and fungi experiments so the table is supported by measured results instead of only descriptions.

The contributions are:
\begin{itemize}
    \item a mechanism-based taxonomy for poisoning and related adversarial methods;
    \item a risk score based on impact, stealth, scalability, and cost-efficiency;
    \item real MNIST and fungi implementations for multiple poisoning, backdoor, and evasion methods;
    \item an implementation audit that distinguishes full implementations from scoped approximations.
\end{itemize}

\section{Related Work}

Classical data poisoning was studied for support vector machines and other learners by Biggio et al.~\cite{biggio2012} and Mei and Zhu~\cite{mei2015}, who formalized poisoning as an optimization problem over training data. Backdoor poisoning became a central model-supply-chain threat with BadNets~\cite{badnets} and targeted backdoor attacks~\cite{targetedbackdoor}. Clean-label poisoning methods such as Poison Frogs~\cite{poisonfrogs} increased stealth by keeping human-visible labels correct. Gradient-matching methods such as Witches' Brew~\cite{witchesbrew} further improved poisoning efficiency by aligning poison gradients with a target objective.

Modern systems introduce additional poisoning surfaces. Web-scale dataset poisoning demonstrates that mutable public data can compromise large training collections~\cite{webscale}. RAG poisoning attacks retrieval corpora rather than model weights~\cite{poisonedrag}. Generative-model poisoning, including Nightshade~\cite{nightshade}, targets text-to-image training data. Backdoor variants such as clean-label backdoors~\cite{cleanlabelbackdoor} and WaNet~\cite{wanet} improve stealth by hiding or warping triggers.

The adversarial-example literature provides related but distinct test-time methods. JSMA uses saliency maps over the model Jacobian~\cite{jsma}; EAD uses an elastic-net optimization objective~\cite{ead}; ZOO estimates gradients with finite differences in black-box settings~\cite{zoo}; HopSkipJump is a query-efficient decision-based attack~\cite{hopskipjump}; Adversarial Patch learns a reusable localized patch~\cite{adversarialpatch}; and SparseFool searches for sparse decision-boundary crossings~\cite{sparsefool}. These methods are useful comparison baselines, but they are not training-time data poisoning by default.

\section{Methodology}

\subsection{Poisoning Definition}

Let a supervised training set be
\begin{equation}
    D = \{(x_i, y_i)\}_{i=1}^{n},
\end{equation}
where \(x_i\) is an input and \(y_i\) is a label. The learner trains parameters \(\theta\) by solving
\begin{equation}
    \theta^\star(D) =
    \arg\min_{\theta}
    \frac{1}{n}\sum_{i=1}^{n}\ell(f_\theta(x_i), y_i)
    + \lambda \Omega(\theta).
\end{equation}
In data poisoning, the attacker modifies the training data:
\begin{equation}
    D' = D_{\mathrm{clean}} \cup D_{\mathrm{poison}},
\end{equation}
so that the resulting model \(\theta^\star(D')\) differs from the clean model \(\theta^\star(D_{\mathrm{clean}})\) in a way that benefits the attacker.

A general poisoning objective can be written as a bilevel problem:
\begin{equation}
\begin{aligned}
    \max_{D_{\mathrm{poison}} \in \mathcal{C}} \quad
        & \mathcal{A}(\theta^\star(D_{\mathrm{clean}} \cup D_{\mathrm{poison}}))
        - c(D_{\mathrm{poison}}) \\
    \mathrm{s.t.} \quad
        & \theta^\star(D_{\mathrm{clean}} \cup D_{\mathrm{poison}})
        =
        \arg\min_{\theta}
        \mathcal{L}(\theta; D_{\mathrm{clean}} \cup D_{\mathrm{poison}}).
\end{aligned}
\end{equation}
Here, \(\mathcal{A}\) is the adversary's objective, \(c\) is a cost or detectability penalty, and \(\mathcal{C}\) captures the attacker's access constraints.

\subsection{Mechanism Categories}

The framework uses mechanism categories as table columns:
\begin{itemize}
    \item G: gradient-based methods;
    \item O: optimization-based methods;
    \item M: minimal or structural changes;
    \item U: universal or transferable methods;
    \item B: backdoor or trigger-based methods;
    \item S: supply-chain and web-scale methods;
    \item R: retrieval or RAG corpus poisoning;
    \item P: preference, reward, or alignment-data poisoning.
\end{itemize}

\subsection{Risk Model}

Operational danger is scored as
\begin{equation}
    \mathrm{DangerScore}=I \times S \times G \times C,
\end{equation}
where \(I\) is impact, \(S\) is stealth, \(G\) is scalability, and \(C\) is cost-efficiency. Each factor is rated from 1 to 5, with 5 indicating greater danger for defenders. Cost-efficiency is high when the attack is cheaper or easier for the adversary.

\section{Experimental Design}

\subsection{MNIST Dataset and Model}

The experiments use MNIST, a real handwritten-digit dataset with 60,000 training images and 10,000 test images~\cite{mnist}. Each input is a \(28 \times 28\) grayscale image. The code trains scikit-learn stochastic-gradient logistic classifiers, giving a transparent baseline whose gradients and decision boundaries are easy to inspect.

\subsection{Fungi Dataset and Transfer Model}

The second experiment uses the local \texttt{fungi} image dataset. The task is binary classification between edible and poisonous fungi, with 360 training images and 90 validation images organized in class folders. This is a more realistic and fine-grained visual task than MNIST. The implementation therefore uses a lazy PyTorch \texttt{Dataset} and \texttt{DataLoader} pipeline rather than loading all images into memory at once.

Images are resized to \(160 \times 160\) RGB. The default model is MobileNetV3-Small with ImageNet pretrained weights~\cite{mobilenetv3}; the feature extractor is frozen and a new classifier head is trained with class-weighted cross entropy. This produces a more realistic transfer-learning threat model than a small from-scratch convolutional network while keeping repeated attack retraining practical on a local machine.

\subsection{Poisoning Implementations}

The comparison suite implements six training-time poisoning or backdoor conditions:
\begin{itemize}
    \item random label flipping;
    \item targeted source-to-target label flipping from digit 7 to digit 1;
    \item subpopulation label poisoning over a structurally selected digit-7 subgroup;
    \item BadNets-style visible patch backdoor;
    \item clean-label patch backdoor over target-class training images;
    \item prototype feature-collision poisoning on MNIST;
    \item true latent-space clean-label Poison Frogs on fungi;
    \item Witches' Brew-style gradient-matching poisoning on fungi.
\end{itemize}

For a backdoor trigger \(T\), the implementation uses a visible lower-right square:
\begin{equation}
    T(x) = (1-m)\odot x + m\odot v,
\end{equation}
where \(m\) is a binary mask and \(v\) is the trigger intensity. The backdoor metric is trigger attack success rate:
\begin{equation}
    \mathrm{ASR} =
    \frac{\sum_{(x,y):y\neq t} \mathbf{1}[\arg\max_k f_\theta(T(x))_k=t]}
    {\sum_{(x,y)} \mathbf{1}[y\neq t]}.
\end{equation}

For subpopulation poisoning, a structural score selects a slice of digit 7:
\begin{equation}
    q(x)=\mathrm{mean}(x_{\mathrm{upper\ right}})-\mathrm{mean}(x_{\mathrm{lower\ left}}).
\end{equation}
Only high-\(q(x)\) source examples are relabeled to the target. The metric is source-subgroup confusion to digit 1.

\subsection{Evasion Implementations}

The suite also implements related test-time attacks against the clean baseline: FGSM, PGD, EAD, JSMA, SparseFool-style sparse boundary crossing, Adversarial Patch, ZOO finite differences, and HopSkipJump-style boundary search. These are included because the original poster used several evasion methods, but the paper labels them as evasion rather than poisoning. The fungi implementation uses an EOT-trained adversarial patch, a vectorized JSMA saliency update, an ISTA-style elastic-net update, and batched finite-difference ZOO coordinates.

For evasion, untargeted success is
\begin{equation}
    \mathrm{USR} =
    \frac{1}{m}\sum_{i=1}^{m}
    \mathbf{1}[\arg\max_k f_\theta(x_i')_k \neq y_i],
\end{equation}
and targeted success toward class \(t\) is
\begin{equation}
    \mathrm{TSR} =
    \frac{1}{m}\sum_{i=1}^{m}
    \mathbf{1}[\arg\max_k f_\theta(x_i')_k = t].
\end{equation}

For the fungi evasion results, success is conditional on the clean input being classified correctly:
\begin{equation}
    \mathrm{CSR} =
    \frac{\sum_i \mathbf{1}[\arg\max f_\theta(x_i)=y_i]
    \mathbf{1}[\arg\max f_\theta(x_i') \neq y_i]}
    {\sum_i \mathbf{1}[\arg\max f_\theta(x_i)=y_i]}.
\end{equation}
This avoids crediting an attack for examples the baseline model already misclassified.

\section{Discussion}

The comparison reveals three lessons. First, backdoors can preserve normal clean accuracy while creating a hidden targeted failure. This makes them more operationally dangerous than simple availability attacks that visibly degrade validation accuracy. Second, subgroup attacks show why aggregate metrics are insufficient; a model can look acceptable overall while failing badly on a selected slice. Third, evasion methods are useful for comparison, but their stage must be labeled correctly. A test-time adversarial example is not data poisoning unless it is used to contaminate training or adaptation data.

The MNIST implementation is intentionally transparent, while the fungi implementation uses a more realistic pretrained vision backbone. Some entries are paper-faithful implementations for the local setting; others are scoped analogs that preserve the core mechanism while avoiding claims about industrial-scale transferability or long-horizon persistence.

\section{Safety and Reproducibility}

The experiments are local, use offline research datasets, and do not target deployed systems, real users, private data, or production services. The repository excludes downloaded MNIST data, the local \texttt{fungi} dataset, and pretrained-weight caches from version control. The main commands are:
\begin{itemize}
    \item \texttt{python scripts/mnist\_poisoning\_experiment.py --download}
    \item \texttt{python scripts/mnist\_attack\_comparison.py --download}
    \item \texttt{python scripts/fungi\_attack\_comparison.py}
\end{itemize}

\section{Conclusion}

This paper presents a risk-based periodic-table framework for data poisoning and related adversarial ML methods. The framework improves the poster by distinguishing poisoning from evasion, assigning operational risk levels, and grounding the discussion in real MNIST and fungi experiments. The results show that backdoor and subpopulation attacks can be especially dangerous because they preserve or obscure aggregate accuracy while creating targeted failures. The fungi experiment also shows the importance of realistic transfer-learning baselines, lazy data loading, and conditional attack-success metrics when moving beyond classroom-scale datasets.
