\begin{abstract}
Data poisoning attacks manipulate training, fine-tuning, retrieval, feedback, or model-supply-chain data so that a learned model behaves incorrectly after deployment. This paper presents a compact, periodic-table-inspired taxonomy matrix that separates training-time poisoning from test-time evasion, groups techniques by attack family, and scores operational danger using impact, stealth, scalability, and cost-efficiency. Unlike a literal Mendeleev layout, every matrix coordinate is functional: rows organize attack surface and operational stage, columns are consistent attack families, and risk is printed inside each cell rather than inferred from cell position. The framework is supported by executable experiments on MNIST, fungi, and fish freshness images. The food-image experiments use lazy PyTorch data loading, a pretrained MobileNetV3-Small transfer model, conditional evasion metrics, all original poster evasion methods, and ten additional poisoning or evasion methods shared during project development. Results show that aggregate clean accuracy is not enough: subpopulation poisoning, clean-label backdoors, and conditional evasion can create severe targeted failures even when ordinary validation metrics look acceptable.
\end{abstract}

\begin{IEEEkeywords}
data poisoning, backdoor attacks, adversarial machine learning, fungi classification, MNIST, AI security

\section{Introduction}

Machine learning systems increasingly depend on data that is scraped, labeled, filtered, retrieved, fine-tuned, or reused across organizations. These data pipelines create an attack surface. A poisoning attacker does not need to break the model code; changing a small portion of the data can change the learned behavior. This project organizes that landscape into a stage-by-family taxonomy matrix for AI security presentations.

The key correction is stage separation. Poisoning methods modify data before or during training, fine-tuning, retrieval, or alignment. Evasion methods such as FGSM, PGD, DeepFool, Carlini-Wagner, Boundary, One-Pixel, UAP, JSMA, ZOO, SparseFool, HopSkipJump, and Adversarial Patch modify inputs at inference time. They are related adversarial ML techniques, but they are not data poisoning unless the adversarial examples are fed back into training or adaptation.

This paper is organized around three research questions:
\begin{itemize}
    \item \textbf{RQ1:} How can data poisoning and related adversarial methods be organized so that mechanism, attack stage, and operational danger are visible at a glance without forcing contradictory visual coordinates?
    \item \textbf{RQ2:} When these methods are implemented on real image data, which attacks create the largest targeted failures under comparable evaluation metrics?
    \item \textbf{RQ3:} What implementation and evaluation choices are necessary to avoid misleading results when moving from a simple benchmark to a realistic fungi image task?
\end{itemize}

The contributions are: (i) a mechanism-and-risk taxonomy for poisoning and related attacks; (ii) implementations for the original poster methods plus ten additional shared methods, audited in \texttt{results/implementation\_coverage.*}; (iii) a transfer-learning fungi experiment that avoids in-memory dataset loading and weak from-scratch baselines; and (iv) reporting artifacts, tables, graphs, and a layout audit for conference presentation.

\section{Related Work}

Classical poisoning was formalized for support vector machines and other learners by Biggio et al.~\cite{biggio2012} and Mei and Zhu~\cite{mei2015}. Backdoor poisoning became prominent with BadNets~\cite{badnets} and targeted backdoors~\cite{targetedbackdoor}. Clean-label attacks such as Poison Frogs~\cite{poisonfrogs} improve stealth by keeping labels correct, while Witches' Brew~\cite{witchesbrew} uses gradient matching to craft efficient poisons. Modern supply-chain and retrieval surfaces include web-scale dataset poisoning~\cite{webscale}, RAG corpus poisoning~\cite{poisonedrag}, and generative-model poisoning such as Nightshade~\cite{nightshade}.

The evasion literature supplies useful comparison methods. DeepFool linearizes the nearest decision boundary~\cite{deepfool}; Carlini-Wagner attacks optimize margin-based adversarial objectives~\cite{cw}; One-Pixel uses differential evolution in a highly sparse search space~\cite{onepixel}; UAP learns an image-agnostic perturbation~\cite{uap}; JSMA computes saliency from the model Jacobian~\cite{jsma}; EAD uses an elastic-net objective~\cite{ead}; ZOO estimates gradients through black-box finite differences~\cite{zoo}; HopSkipJump is decision-based~\cite{hopskipjump}; Adversarial Patch learns a reusable localized perturbation~\cite{adversarialpatch}; and SparseFool searches for sparse decision-boundary crossings~\cite{sparsefool}.

\section{Framework}

Let \(D=\{(x_i,y_i)\}_{i=1}^{n}\) be a clean training set and let \(\theta^\star(D)\) be the model trained by minimizing empirical loss. Poisoning replaces \(D\) with \(D'=D_{\mathrm{clean}}\cup D_{\mathrm{poison}}\). A general poisoning objective is
\begin{equation}
\begin{aligned}
 \max_{D_{\mathrm{poison}}\in\mathcal{C}}\quad
   & \mathcal{A}(\theta^\star(D_{\mathrm{clean}}\cup D_{\mathrm{poison}}))
   - c(D_{\mathrm{poison}})\\
 \text{s.t.}\quad
   & \theta^\star=\arg\min_{\theta}\mathcal{L}(\theta;D_{\mathrm{clean}}\cup D_{\mathrm{poison}}),
\end{aligned}
\end{equation}
where \(\mathcal{A}\) is the attacker's objective and \(\mathcal{C}\) encodes access constraints. The table columns are attack-family columns: label manipulation, feature/input manipulation, clean-label poisoning, backdoor/trigger poisoning, gradient/model-update poisoning, influence/data-selection poisoning, federated/distributed poisoning, generative/synthetic-data poisoning, preference/RLHF poisoning, retrieval/RAG poisoning, supply-chain poisoning, and dynamic/adaptive poisoning.

Operational danger is scored as
\begin{equation}
\mathrm{DangerScore}=I\times S\times G\times C,
\end{equation}
where \(I\) is impact, \(S\) is stealth, \(G\) is scalability, and \(C\) is attacker cost-efficiency. Risk levels are R1: 400--625, R2: 200--399, R3: 100--199, and R4: below 100. Figure~\ref{fig:periodic} is the paper-readable visual artifact. Each compact cell entry contains a short symbol and risk label; the poster/slide version additionally uses element-like cells with index, symbol, attack name, and risk/family label. The layout deliberately abandons literal chemical-table geometry. Rows represent attack surface or operational stage, columns use one consistent family dimension, and inference-time evasion analogs are separated from poisoning attacks. Glaze is retained as a defensive poisoning-like cloak, not as an ordinary malicious poisoning attack. Numbers are reference identifiers only, not chronology or severity ranking.

\section{Experimental Design}

\subsection{Datasets}

MNIST provides a transparent benchmark with 60,000 training and 10,000 test handwritten digits~\cite{mnist}. The main visual-security experiment uses a local fungi dataset with two classes: edible and poisonous. A second food-safety dataset was added from the FoodDDP fish freshness setup: local folders \texttt{fresh2} and \texttt{non-fresh2} contain 40 horse-mackerel images total, and the safety-critical direction is non-fresh fish classified as fresh. Because the fish folder is flat rather than split into \texttt{train/val}, the runner creates a deterministic stratified 75/25 split. \subsection{Model and Metrics}

The fungi model is MobileNetV3-Small with ImageNet pretrained weights~\cite{mobilenetv3}. The feature extractor is frozen and a new classifier head is trained with class-weighted cross entropy. This transfer-learning setup is more realistic for fine-grained fungi recognition than a small from-scratch convolutional network. Images are resized to \(160\times160\) RGB.

For poisoning and backdoors, the model is retrained on modified training data and evaluated on clean validation data plus an attack-specific metric. For evasion, success is conditional on the clean model first classifying the image correctly:
\begin{equation}
\mathrm{CSR}=
\frac{\sum_i \mathbf{1}[f(x_i)=y_i]\mathbf{1}[f(x_i')\neq y_i]}
{\sum_i \mathbf{1}[f(x_i)=y_i]}.
\end{equation}
This prevents inflated attack success caused by baseline errors.

\subsection{Implementation Corrections}

The fungi code addresses common pitfalls. It uses a custom \texttt{Dataset} and \texttt{DataLoader} for out-of-core batching. Clean-label Poison Frogs is implemented as latent feature optimization, minimizing feature distance while projecting the poison image back into an \(L_\infty\) ball around the base image. The poster methods are implemented as FGSM, PGD, DeepFool L2, Carlini-Wagner L2, Boundary/HopSkipJump search, One-Pixel differential evolution, and UAP. EAD uses a formal ISTA proximal solver: a gradient step on the smooth targeted objective followed by soft-thresholding for the \(L_1\) term and projection into the box and \(L_\infty\) constraints. JSMA saliency is vectorized across a batch. ZOO batches finite-difference coordinate probes. The adversarial patch is trained with random-location expectation over transformation.

The highest fungi evasion effects are PGD, DeepFool, and UAP, each reaching 1.0000 conditional untargeted success. JSMA remains a strong sparse targeted method at 0.9375, while Carlini-Wagner reaches 0.7429 targeted ASR. The strongest poisoning-style selected-slice result is subpopulation poisoning, while the strongest trigger result is the gradient-aligned Sleeper-Agent-style backdoor. Poison Frogs increases the safety-critical poisonous-to-edible confusion above the clean baseline while preserving clean-label constraints. The Witches' Brew-style run demonstrates gradient alignment, but its measured effect is weaker on this small validation split than the subgroup and backdoor attacks.

The fish freshness extension uses the same method table on a much smaller split: 30 training images and 10 validation images. The clean baseline reaches 0.8000 accuracy with 0.0000 non-fresh-to-fresh confusion. The top poisoning/backdoor result is the Sleeper-Agent-style backdoor at 0.8000 stealth-trigger ASR. The strongest fish evasion results are FGSM, PGD, DeepFool, Carlini-Wagner, and UAP at 1.0000 conditional success, while EAD and Boundary/HopSkipJump reach 0.8000 targeted ASR. \subsection{Answers to Research Questions}

\textbf{RQ1:} The taxonomy matrix answers the organization question by making every coordinate auditable: the vertical coordinate is the attack surface or operational stage, the horizontal coordinate is a consistent attack-family column, and the panel determines whether a method is poisoning proper or an evasion/transfer analog. Risk remains visible inside each cell but no longer drives a visually misleading ``extreme'' row. This removes arbitrary chemical-table gaps and separates related evasion methods from data-poisoning methods.

\textbf{RQ2:} On the fungi task, the largest evasion failures are PGD, DeepFool, and UAP with 1.0000 conditional untargeted success; JSMA reaches 0.9375 conditional targeted ASR and Carlini-Wagner reaches 0.7429. On the fish task, FGSM, PGD, DeepFool, Carlini-Wagner, and UAP reach 1.0000 conditional success, while the Sleeper-Agent-style backdoor reaches 0.8000 stealth-trigger ASR. These results answer RQ2 by showing that inference-time methods can be strongest under conditional evaluation, while subgroup and backdoor poisoning create the most important training-time targeted failures.

\textbf{RQ3:} The experiment answers the implementation question by replacing in-memory image stacking with lazy loading, replacing a weak from-scratch CNN with pretrained MobileNetV3-Small transfer learning, using true latent-space feature collision instead of pixel blending, adding explicit poster-method implementations, vectorizing JSMA, batching ZOO finite-difference queries, and reporting conditional attack success only on clean-correct examples.

\section{Discussion}

The results support three security lessons. First, aggregate accuracy is not a sufficient safety metric. The clean fungi baseline already confuses some poisonous samples as edible, and several attacks worsen a targeted failure mode without simply maximizing visible accuracy collapse. Second, attack stage matters. Evasion methods can be dramatic, but they should not be described as poisoning unless they enter the training or adaptation data. Third, realistic baselines matter: pretrained transfer learning, lazy loading, and conditional success metrics make the fungi experiment much closer to a real AI safety presentation than a purely toy implementation.

The implementation audit is intentionally conservative. The code reproduces the core mechanism of all poster methods plus ten additional shared methods: Poison Frogs, Sleeper-Agent-style backdoor, Adversarial Patch, Witches' Brew-style gradient matching, JSMA, ZOO, SparseFool-style sparse boundary search, EAD, subpopulation poisoning, and BadNets. It does not claim industrial-scale transferability, full physical-world patch robustness, or long-horizon LLM sleeper-agent persistence.

\section{Conclusion}

This paper answers the three research questions as follows. First, data poisoning can be organized as an auditable stage-by-family taxonomy matrix by separating attack surface, family, risk, and poisoning-versus-evasion status. Second, the fungi and fish experiments show that PGD, DeepFool, UAP, JSMA, and Carlini-Wagner can produce the strongest evasion failures, while subpopulation poisoning and Sleeper-Agent-style backdoors produce the strongest training-time targeted failures. Third, realistic conclusions require careful engineering: lazy loading, transfer learning, latent-space poisoning, explicit implementation coverage, vectorized/batched attacks, and conditional success metrics. The final framework is suitable for a short IEEE-style AI security conference paper and for an at-a-glance presentation of operational risk.

\section*{Reproducibility and Safety}

The experiments are local and do not target deployed systems, private data, or production services. Main commands are:
\texttt{python scripts/mnist\_attack\_comparison.py --download},
\texttt{python scripts/fungi\_attack\_comparison.py}, and
\texttt{python scripts/generate\_reporting\_artifacts.py}.
The generated table-layout audit should contain only \texttt{consistent} statuses. The \texttt{fungi/} dataset and pretrained-weight cache are excluded from version control.
